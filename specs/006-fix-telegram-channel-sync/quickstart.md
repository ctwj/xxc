# Quickstart: 验证 TelegramSync 频道消息同步修复

**Feature**: 006-fix-telegram-channel-sync
**Date**: 2026-05-09

## 前提条件

1. 系统已部署，TelegramSync 插件已安装
2. 已通过管理后台完成 Telegram 账号认证
3. 至少有一个可用于测试的 Telegram 广播频道（broadcast channel）

## 快速验证步骤

### 1. 启动开发环境

```bash
task dev
```

### 2. 配置频道监听

1. 进入管理后台 → 插件 → TelegramSync
2. 点击"获取频道列表"，确认能看到广播频道（`is_broadcast: true`）
3. 选择要监听的广播频道，启用监听
4. 保存配置

### 3. 验证频道消息同步

1. 在已绑定的 Telegram 广播频道中发布一条测试消息
2. 等待 60 秒
3. 检查 CMS 后台文章列表，应出现新文章
4. 确认文章内容与频道消息一致

### 4. 验证群组消息不受影响

1. 在已绑定的 Telegram 群组中发送一条测试消息
2. 确认仍然正常生成文章

### 5. 验证配置完整性

1. 配置 3 个频道：2 个启用，1 个禁用
2. 在启用频道发送消息，触发消息处理
3. 刷新频道配置页面，确认 3 个频道配置全部存在

### 6. 检查日志

查看日志中以下关键信息：
- `updates.Manager 已启动` — 确认 Manager 正常启动
- `handleNewChannelMessage 被调用` — 确认频道消息回调被触发
- `找到频道配置` — 确认频道 ID 匹配成功
- 文章创建成功的日志

## 回归测试检查清单

- [ ] 广播频道文本消息 → 生成文章
- [ ] 广播频道图片消息 → 生成文章含图片
- [ ] 广播频道转发消息 → 正常生成文章
- [ ] 群组消息 → 继续正常工作
- [ ] 多频道同时监听 → 全部正常
- [ ] 禁用的频道 → 不生成文章，配置不丢失
