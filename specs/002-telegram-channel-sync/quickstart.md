# Quickstart: Telegram Channel Sync

**Date**: 2026-04-30
**Feature**: 002-telegram-channel-sync

## 前置条件

1. **Telegram API 凭证**
   - 访问 https://my.telegram.org/apps 创建应用
   - 获取 `App ID` 和 `App Hash`

2. **Telegram 账号**
   - 需要一个有效的手机号用于认证
   - 账号需要已加入要监听的频道

3. **Moss CMS**
   - 确保 Moss 后端正常运行
   - 确保数据库连接正常

## 安装步骤

### 1. 添加依赖

```bash
cd main
go get github.com/gotd/td@latest
go mod tidy
```

### 2. 注册插件

编辑 `main/startup/startup.go`，添加插件注册：

```go
func initPlugins() {
    appService.PluginInit(
        // ... 现有插件 ...
        plugins.NewTelegramChannelSync(),
    )
}
```

### 3. 初始化数据库

插件首次加载时会自动创建所需的数据表。

### 4. 重启服务

```bash
# 开发模式
task dev

# 或生产模式
task run
```

## 配置指南

### 1. 配置 Telegram API

访问管理后台 `/admin/telegram-sync/config`：

1. 填写 `App ID` 和 `App Hash`
2. 填写手机号（国际格式，如 `+8613800138000`）
3. 点击"保存配置"

### 2. Telegram 认证

1. 点击"发送验证码"
2. 在 Telegram 中查收验证码
3. 输入验证码完成认证

认证成功后，会话会被加密保存，后续重启无需重新认证。

### 3. 添加监听频道

访问管理后台 `/admin/telegram-sync/channels`：

1. 点击"添加频道"
2. 填写频道信息：
   - **频道 ID**: Telegram 频道 ID（负数，如 `-1001234567890`）
   - **频道名称**: 显示名称
   - **目标分类**: 同步文章的分类
   - **过滤规则**: 可选，设置关键词过滤

### 4. 获取频道 ID

有几种方式获取频道 ID：

**方法 1: 使用 @userinfobot**
1. 在 Telegram 中搜索 `@userinfobot`
2. 将其添加到目标频道
3. 它会自动发送频道 ID

**方法 2: 从频道链接解析**
- 公开频道链接格式: `https://t.me/c/1234567890/1`
- 频道 ID 为: `-100` + `1234567890` = `-1001234567890`

**方法 3: 使用 API 查询**
- 通过 `resolveUsername` API 解析频道用户名

## 过滤规则配置

### 关键词白名单

只同步包含指定关键词的消息：

```json
{
  "type": "whitelist",
  "keywords": ["技术", "教程", "编程"],
  "case_sensitive": false
}
```

### 关键词黑名单

排除包含指定关键词的消息：

```json
{
  "type": "blacklist",
  "keywords": ["广告", "推广", "赞助"],
  "case_sensitive": false
}
```

### 消息类型过滤

在"消息类型"字段中设置，多个类型用逗号分隔：

- `text` - 纯文本消息
- `photo` - 图片消息
- `video` - 视频消息
- `document` - 文档消息

示例: `text,photo` 表示只同步文本和图片消息。

### 长度过滤

- **最小长度**: 消息内容的最小字符数
- **最大长度**: 消息内容的最大字符数（超过会被截断）

## 运维指南

### 监控连接状态

访问 `/admin/telegram-sync/status` 查看：
- 连接状态
- 认证状态
- 监听频道数量
- 运行时长

### 查看同步日志

访问 `/admin/telegram-sync/logs` 查看：
- 同步成功/失败记录
- 错误信息
- 消息摘要

### 处理认证过期

如果会话过期：
1. 系统会自动检测并标记
2. 访问配置页面会提示需要重新认证
3. 重新发送验证码并验证

### 日志清理

插件会自动清理过期的同步日志（根据配置的保留天数）。

## 故障排除

### 问题: 无法连接到 Telegram

**可能原因**:
- 网络问题
- 防火墙阻止
- Telegram 服务不可用

**解决方案**:
- 检查网络连接
- 配置代理（如需要）
- 查看日志获取详细错误

### 问题: 认证失败

**可能原因**:
- 验证码错误或过期
- 手机号格式不正确
- 账号被限制

**解决方案**:
- 确保手机号格式正确（国际格式）
- 重新发送验证码
- 检查账号状态

### 问题: 消息未同步

**可能原因**:
- 频道未启用
- 消息被过滤规则排除
- 网络中断

**解决方案**:
- 检查频道状态是否为"启用"
- 检查过滤规则配置
- 查看同步日志确认原因

### 问题: 图片未下载

**可能原因**:
- 未启用媒体下载
- 图片大小超过限制
- 存储空间不足

**解决方案**:
- 检查"下载媒体"配置
- 调整"最大图片大小"限制
- 检查存储空间

## 安全建议

1. **保护 API 凭证**
   - 不要在代码中硬编码 App ID 和 App Hash
   - 通过管理界面配置

2. **保护会话数据**
   - 设置强加密密钥
   - 定期检查会话状态

3. **限制监听频道**
   - 只监听必要的频道
   - 定期审查频道列表

4. **监控异常行为**
   - 关注错误日志
   - 设置告警通知
