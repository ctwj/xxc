# Implementation Plan: 修复 TelegramSync 频道消息同步

**Branch**: `006-fix-telegram-channel-sync` | **Date**: 2026-05-09 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/006-fix-telegram-channel-sync/spec.md`

## Summary

TelegramSync 插件在群组中正常工作但在广播频道中不产生文章。根本原因是 `updates.Manager` 缺少频道的 access hash 配置，导致 `UpdateNewChannelMessage` 被静默丢弃。修复方案：实现自定义 `ChannelAccessHasher`，从频道配置中提供 access hash，同时修复频道配置被意外覆盖的 bug。

## Technical Context

**Language/Version**: Go 1.x
**Primary Dependencies**: gotd/td v0.108.0, Fiber (gofiber/fiber/v2), GORM

## Constitution Check

Constitution 文件为空模板，无约束。跳过 gate 检查。

## Root Cause

gotd/td 的 `updates.Manager` 在处理 `UpdateNewChannelMessage` 时，需要频道的 access hash 来创建 channel state。当前插件仅设置了 `Handler` 而未配置 `AccessHasher`，导致默认的内存 AccessHasher 为空，频道更新被静默丢弃。

详见 [research.md](research.md)。

## Implementation Steps

### Step 1: 实现 `channelAccessHasher`

**File**: `main/plugins/telegram_sync/client.go`

在 `client.go` 中新增 `channelAccessHasher` 结构体：

```go
type channelAccessHasher struct {
    hashes map[int64]int64
    mu     sync.RWMutex
}

func newChannelAccessHasher() *channelAccessHasher {
    return &channelAccessHasher{
        hashes: make(map[int64]int64),
    }
}

func (h *channelAccessHasher) GetChannelAccessHash(ctx context.Context, userID int64, channelID int64) (int64, bool, error) {
    h.mu.RLock()
    defer h.mu.RUnlock()
    hash, ok := h.hashes[channelID]
    return hash, ok, nil
}

func (h *channelAccessHasher) SetChannelAccessHash(ctx context.Context, userID int64, channelID int64, hash int64) error {
    h.mu.Lock()
    defer h.mu.Unlock()
    h.hashes[channelID] = hash
    return nil
}
```

### Step 2: 配置 `AccessHasher` 到 `updates.Config`

**File**: `main/plugins/telegram_sync/client.go`

修改 `NewClientWithStorage` 方法，将 `channelAccessHasher` 配置到 `updates.Config`：

```go
func NewClientWithStorage(config *ClientConfig, storage *DBStorage, log *zap.Logger) (*Client, error) {
    // ... existing code ...

    c.accessHasher = newChannelAccessHasher()

    c.updatesMgr = updates.New(updates.Config{
        Handler:      c.dispatcher,
        AccessHasher: c.accessHasher,  // NEW
    })

    // ... existing code ...
}
```

同时在 `Client` 结构体中添加 `accessHasher` 字段。

### Step 3: 在频道配置时写入 access hash

**File**: `main/plugins/TelegramChannelSync.go`

在获取频道列表（`GetUserChannels`）和保存频道配置时，将 access hash 写入 `accessHasher`：

- 在 `handleChannelMessage` 中，收到消息处理前，确保频道的 access hash 已缓存
- 在 `SaveChannels` 后，更新 access hash 缓存
- 在客户端初始化后，从已配置的频道中加载 access hash

### Step 4: 修复 `p.channels` 覆盖 bug

**File**: `main/plugins/TelegramChannelSync.go`

将 `handleChannelMessage` 中的：

```go
p.channels = enabledChannels  // BUG: 覆盖了完整列表
```

改为仅用于过滤查询，不修改原始列表：

```go
// 仅获取启用频道用于过滤，不覆盖完整列表
var enabledChannels []telegram_sync.ChannelConfig
for _, ch := range p.channels {
    if ch.Status == 1 {
        enabledChannels = append(enabledChannels, ch)
    }
}
// 删除: p.channels = enabledChannels
```
